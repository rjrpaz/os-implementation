In class we have discussed how running programs (and the kernel) use a stack of activation records to store the parameters, local variables, temporary data, and return addresses associated with procedure calls.

Each time you invoke a procedure (i.e., call a function or method), a new activation record is created for that call.  A special processor register, the stack pointer, keeps track of which activation record is being used, and also points to a specific data item within the activation record.  On x86 CPUs, this is the esp register.  Sometimes, programs may use an additional register, the frame pointer, to point to the current activation record.  On x86 CPUs, this is the ebp register.  However, the frame pointer is not strictly necessary, and many compilers simply use it as an extra general-purpose register.

Usually, the activation record stack grows down in memory, from higher addresses to lower addresses.
